#含有无符号类型的表达式

尽管我们不会故意给无符号对象赋一个负值，却可能（特别容易）写出这么做的代码。

##情况1
例如，当一个算术表达式中既有无符号数又有int值时，那个int值就会转换成无符号数。把int转换成无符号数的过程和把int直接赋给无符号变量一样：

unsigned u=10;
int i=-42;
std::cout<<i+i<<std::endl;//输出-84
std::cout<<u+i<<std::endl;//如果int占32位，输出4294967264

第一个表达式里，两个负整数想加并得到了期望的结果。在第二个输出表达式里，相加前首先把整数-42转换成无符号数。把负数转换成无符号数类似于直接给无符号数赋一个负值，结果等于给这个负数加上无符号数的模。

##情况2
当从无符号数中减去一个值时，不管这个值是不是无符号数，我们都必须确保结果不能是一个负值：

unsigned u1=42,u2=10;
 std::cout<<u1-u2<<std::endl;//正确:输出32
 std::cout<<u2-u1<<std::endl;//正确不过是取模后的值。

##情况3
无符号数不会小于0这一事实关系到循环的写法。例如：

for(int i=10;i>=0;--i)
    std::cout<<i<<std::endl;

可能你会觉得反正也不打算输出负数，可以用无符号数来重写这个循环。然而，这个不经意的改变却意味着死循环。

//错误:变量u永远也不会小于0，循环条件一直成立
for(unsigned u=10;u>=0;--u)
    std::cout<<u<<std::endl;

当u=0时，然后继续执行for语句里的表达式，表达式–u从u当中减去1，得到-1并不满足无符号数的要求，被自动转换。
一种解决的办法是，用while语句来代替for语句，因为前者让我们能够在输出变量之前（而非之后）先减去1：

unsigned u=11;//确定要输出的最大数，从比它大1的数开始
while(u>0){
    --u;//先减去1，这样最后一次迭代就会输出0
    std::cout<<u<<std::endl;
}
